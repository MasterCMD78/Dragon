13. Security

Every admin action should write to an admin_logs table (or the existing production table if available), including:

* Admin Telegram ID
* Action
* Target user
* Amount (if applicable)
* Timestamp
* Reason

This gives you a full audit trail.

⸻

14. Future Moderation Tools

* Suspend user
* Unsuspend user
* Delete spam task submissions
* Reset referral abuse
* Flag suspicious accounts

⸻

Safety Rules

The admin panel should:

* Require is_admin = true on every endpoint.
* Never expose admin routes to regular users.
* Require confirmation for destructive actions.
* Never modify data directly outside the normal application flow.
* Record every sensitive action in the audit log.
