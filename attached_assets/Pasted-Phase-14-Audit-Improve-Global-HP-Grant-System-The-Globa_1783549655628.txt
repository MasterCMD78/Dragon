Phase 14 – Audit & Improve Global HP Grant System

The Global HP Grant feature has been implemented. Before deployment, perform a complete audit and improve it to production quality.

1. Security

* Ensure only authenticated admins can access the endpoint.
* Verify every request passes requireAdmin.
* Prevent non-admins from calling the API.
* Validate amount and reason on both frontend and backend.

2. Confirmation Protection

Improve the confirmation flow.

Instead of only showing:

“Are you sure?”

Require the admin to type:

GRANT

before the Grant button becomes active.

This prevents accidental mass grants.

⸻

3. Maximum Grant Limit

Prevent economy abuse.

Rules:

* Maximum HP per grant = 5,000
* Minimum HP = 1

Reject any value outside the range.

⸻

4. Cooldown

Prevent multiple accidental grants.

After a successful grant:

* Disable the Grant button for 60 seconds.
* Display a countdown timer.

⸻

5. Duplicate Protection

Prevent sending the exact same grant twice.

If another grant with the same:

* amount
* reason

was sent within the last 10 minutes,

reject it with a friendly error.

⸻

6. Preview Screen

Before confirming, display:

* Number of users receiving HP
* Total HP to be distributed
* Estimated new circulating HP

Example:

Users: 412

Amount per user: 100 HP

Total Distribution:
41,200 HP

Require confirmation after preview.

⸻

7. Notifications

Ensure notifications are sent only after the transaction commits successfully.

A notification failure must never rollback the grant.

⸻

8. Admin Log

Log:

* Admin ID
* Admin name
* Amount
* Total users
* Total HP distributed
* Reason
* Timestamp
* IP address (if available)

⸻

9. Rollback Support

Implement an emergency rollback.

Admin should be able to reverse the last global grant only.

Requirements:

* Restore balances.
* Reverse transaction records.
* Log rollback action.
* Prevent rolling back twice.

⸻

10. Performance

If the user count grows:

* Process updates in batches.
* Avoid memory spikes.
* Keep everything inside a transaction where appropriate.

⸻

11. Statistics

After a grant, automatically refresh:

* Total users
* Total HP
* Total transactions
* Recent admin logs

without reloading the page.

⸻

12. Testing

Run:

* TypeScript check
* Production build
* End-to-end tests
* Verify zero errors

Return a report containing:

* Bugs found
* Bugs fixed
* Performance improvements
* Security improvements
* Production readiness
